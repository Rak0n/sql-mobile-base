Questa guida riassume tutta la sintassi necessaria per l'estrazione e l'analisi dei dati.

1. L'Ordine Sacro delle Clausole

SQL è una lingua gerarchica. Non puoi cambiare l'ordine di questi comandi:

  1. SELECT : Cosa vuoi vedere (colonne o calcoli).
  2. FROM : La tabella principale di partenza.
  3. JOIN : Altre tabelle da collegare (usando ON).
  4. WHERE : Filtri sulle righe singole (es. temperatura > 500).
  5. GROUP BY : Raggruppamento (obbligatorio se usi SUM, AVG, COUNT).
  6. HAVING : Filtro sui gruppi (es. "mostra solo gruppi con media > 10").
  7. ORDER BY : Ordinamento finale (ASC o DESC).
  8. LIMIT : Quante righe mostrare (es. LIMIT 5).

📊 3. Funzioni di Aggregazione (Verticali)

Si usano per fare statistiche su intere colonne.

  • COUNT(*) : Conta quante righe ci sono.
  • SUM(colonna) : Somma totale.
  • AVG(colonna) : Media.
  • MIN(colonna) / MAX(colonna) : Valore minimo o massimo.

🔗 4. Il Potere dei JOIN

  • INNER JOIN : Mostra solo le righe che hanno una corrispondenza in entrambe le tabelle.
  • LEFT JOIN : Mostra tutte le righe della tabella a sinistra (quella nel FROM), anche se non trovano nulla in quella a destra.

🧪 5. Manipolazione Dati e Funzioni "Cool"

Concatenazione (Incollare testi): Usa || per unire colonne e testo fisso.

SELECT nome_esteso || ' [ID: ' || id || ']' AS etichetta
-- Risultato: "Wood Chips [ID: FS_001]"

Logica Condizionale (Il "SE" di Excel): Usa CASE WHEN per creare categorie.

SELECT id_run,
  CASE 
    WHEN resa_olio > 45 THEN 'Alta'
    WHEN resa_olio > 30 THEN 'Media'
    ELSE 'Bassa'
  END AS valutazione_resa
FROM runs_pirolisi;

Matematica Semplice: Puoi usare +, -, *, / tra colonne.

SELECT (resa_olio / resa_totale) * 100 AS percentuale_olio

💡 6. La Regola d'Oro del GROUP BY

Se nella SELECT hai una colonna "normale" e una "calcolata", la normale deve essere ripetuta nel GROUP BY.

SELECT categoria, AVG(c_mean) -- categoria è normale
FROM feedstocks
GROUP BY categoria; -- quindi va qui!
